1 接入概览

统一服务平台采取"住建主导、贝壳承建、同业开放"原则。机构通过本 API 推送房源与运营记录，平台完成数据清洗、价格红线校验、合规验真后统一上架；住建后台只读监管，数据接受省社科院/东南大学/省房协月度审计。

三种接入模式

模式一自营接入

适用：具备完整运营能力的机构（自如 / 龙湖 / 华润等）。

仅推送房源与备案数据；运营、收款、维修由机构闭环。

调用范围：/listings/* + /filings/*

模式二总代监管

适用：国资 / 城投有招租能力但合规需平台兜底。

平台代收代付与三方备案；机构持有经营权。

调用范围：以上 + /payments/* + /escrow/*

模式三深度托管

适用：业主无运营能力 / 信托垫资项目。

平台获完整经营权（招租 + 装改 + 收款 + 维修）。

调用范围：仅 /assets/*（业主侧资产推送即可）

2 认证授权

2.1 资质前置

仅向以下主体颁发凭证：

  • 住建厅"运营主体白名单"在册机构（≥A 级）
  • 省级国资 / 城投及其授权代理
  • 金融机构（银行 / 信托）持有的保租房项目方

申请入口：运营主体白名单 → 申请接入，材料齐全后 5 个工作日完成审核。

2.2 OAuth 2.0 (client_credentials)

POST /oauth/token 获取 access_token

Content-Type: application/x-www-form-urlencoded

grant_type=client_credentials
&client_id=ziroom_jiangsu_2026
&client_secret=****

}

2.3 请求签名（敏感写操作）

除 Bearer Token 外，写接口需要附加 X-Signature（HMAC-SHA256），算法：

signature = HMAC_SHA256(
  client_secret,
  HTTP_METHOD + "\n" + PATH + "\n" + X-Timestamp + "\n" + sha256(body)
)

请求头示例：

Authorization: Bearer eyJhbGciOi...
X-Timestamp: 1717574400
X-Nonce: a3f8c2-...
X-Signature: sha256=8a1f...
X-Operator-Id: ziroom_jiangsu_2026

3 房屋数据字典

字典由住建厅主导制定，分为静态披露字段（房屋本体属性）与动态运营记录（生命周期事件）。所有接入方按本字典推送，平台清洗后统一展示。

3.1 静态字段（Listing）

字段类型必填说明
external_idstring是机构内部唯一房源 ID
operator_idstring是白名单中的机构标识
project_namestring是项目 / 小区名（中文）
addressobject是{ city, district, street, building, room }
geoobject是WGS84 { lng, lat }
layoutstring是枚举：1B1H 2B1H 3B2H STUDIO
area_sqmnumber是建筑面积，保留 1 位小数
orientationstring否N/S/E/W/NE/SE/SW/NW
floorobject是{ current, total }
rent_monthlynumber是单位元/月，须 ≤ 价格红线
service_fee_pctnumber是服务费率，省级标准 ≤ 8%
market_price_refnumber否市场参考价（用于展示对照）
house_qualityobject是好房子 4 维度评级 — 见 3.3
amenitiesstring[]否AC / FRIDGE / WASHER / WIFI / SMART_LOCK / ELEVATOR
photosobject[]是≥ 6 张，详见 3.4
access_modestring是SELF(自营) / AGENCY(总代) / TRUSTEE(托管)
cert_nostring是保租房项目认定文号（住建口径）

3.2 动态运营记录（Event）

事件类型触发时机必传字段
LISTED上架—
VIEWING_BOOKED预约带看tenant_hash, scheduled_at
CONTRACT_SIGNED签约filing_no, term_months, deposit, monthly_rent
PAYMENT_SETTLED支付到账payment_channel, amount, due_at
REPAIR_REQUESTED报修category, severity, sla_hours
REPAIR_CLOSED维修完成cost, satisfaction_score
CHECKOUT退租reason, return_amount
DELISTED下架reason

3.3 好房子四维度评级

{
  "house_quality": {
    "comfort":  4,   // 舒适：独卫、采光、隔音
    "green":    3,   // 绿色：节能门窗、空气循环
    "smart":    5,   // 科技：智能门锁、智能电表、HDR 监控
    "safety":   4    // 安全：消防、应急通道、保险覆盖
  }
}

评级 1–5；总星 = round(avg)。住建厅季度抽检，连续两季度低于 3 星将被强制下架整改。

3.4 图片资源

推送时使用对象引用，平台拉取后 CDN 化：

{
  "photos": [
    { "url": "https://cdn.ziroom.com/...", "tag": "LIVING_ROOM", "order": 1 },
    { "url": "https://cdn.ziroom.com/...", "tag": "BEDROOM_MASTER", "order": 2 }
  ]
}

分类标签：LIVING_ROOM / BEDROOM_MASTER / BEDROOM_SECOND / KITCHEN / BATHROOM / BALCONY / EXTERIOR / FLOORPLAN

4.5 三方备案接口

POST /v1/filings 提交租赁合同至住建备案系统

平台代为对接住建厅备案系统；机构推送即视为已尽备案义务，回执 5 分钟内返回。

5 价格红线校验

每条房源在 batchUpsert 时同步校验，校验源 = 江苏住房租赁数据治理委员会发布的"区域 × 户型 × 季度"上限表。

GET /v1/redline?city=南京市&district=江宁区&layout=1B1H 查询当前红线 
{
  "city": "南京市",
  "district": "江宁区",
  "layout": "1B1H",
  "redline_monthly": 3400,
  "market_avg": 3120,
  "updated_at": "2026-06-01",
  "audit_by": ["清华大学","东南大学","省社科院"]
}
注：价格红线每季度由数据治理委员会调整一次；机构如对红线有异议，可通过 /v1/redline/disputes 提交申诉，60 天内由第三方审计组裁决。

6 备案回流 Webhook

住建系统回执由平台中转至机构端。机构在控制台配置 callback URL，平台用与请求签名相同的 HMAC 算法签名后投递，3 次重试。

事件类型

event说明
listing.approved房源通过住建审核，已上架
listing.rejected审核未通过
filing.confirmed合同备案号已生效
redline.updated所辖区域红线发生变化（建议存量复核）
compliance.alert合规预警（连续投诉 / 评分过低 / 违规收费）

Payload 示例

POST https://ziroom.com/hooks/bzf-jiangsu
X-BZF-Signature: sha256=...

{
  "event": "filing.confirmed",
  "occurred_at": "2026-06-05T14:35:12+08:00",
  "data": {
    "listing_id": "L-JS-NJ-9f3a...",
    "filing_no": "NJ-FL-2026-0010088",
    "valid_until": "2027-06-04"
  }
}

7 错误码

HTTPcode含义处置
400E_VALIDATION字段格式错误按返回 details 修正
400E_PRICE_REDLINE租金超红线调整租金，或 /redline/disputes 申诉
400E_PHOTO_INSUFFICIENT图片少于 6 张补图后重推
400E_CERT_NO_MISSING保租房认定文号缺失或失效联系区住建局核发
401E_AUTHtoken 失效重取 token
403E_SCOPEscope 不足升级机构资质
409E_DUPLICATE_EXTIDexternal_id 与已有冲突使用 PUT 更新
422E_HOUSE_QUALITY_LOW好房子评级 < 3整改后重报
429E_RATE_LIMIT触发限流退避重试，或申请升级
503E_REDLINE_AUDIT红线接口审计中稍后 5 分钟重试

8 接入 SLA 与流程

承诺 6–8 周完成接入。流程：

  1. Week 0–1：资质审核（白名单备案 / 网安等保 / 数据安全协议）
  2. Week 2：颁发沙箱凭证 + 1V1 技术对接群
  3. Week 3–4：字典映射 + 接口联调 + 单元测试
  4. Week 5：UAT 全链路演练（含 redline / webhook / 备案回流）
  5. Week 6：合规审计 + 隐私评估（PIA）+ 安全渗透测试
  6. Week 7–8：灰度（10% 房源）→ 全量切流 → 住建备案

运行期 SLA

指标目标值违约赔付
接口可用性（生产）≥ 99.9% / 月承建方付月费 5% / 0.1%
批量推送 P99 延迟≤ 1500 ms—
Webhook 投递成功率≥ 99%（3 次重试内）失败队列保留 7 天
备案回执时延≤ 5 min（住建系统时延除外）—

9 Excel 模板（非 API 渠道）

面向尚未具备 API 能力的小型机构 / 国资项目方，提供统一 Excel 模板（与上述字典字段 1:1 对应）。机构填写后上传，平台后台批量导入。

  • 下载模板：listings-template-v1.2.xlsx
  • 导入入口：运营主体白名单 → 数据导入
  • 单次最多 5000 行；导入后 10 分钟内邮件返回校验报告。
过渡安排：Excel 渠道仅限接入首年使用，第二年起强制 API 化（豁免：年存量 < 200 套的微型机构）。

归口部门：江苏省住房和城乡建设厅 · 住房租赁监管处

承建方：贝壳找房（保租房统一服务平台）

审计与公示：清华大学公共管理学院 · 东南大学 · 江苏省社科院 · 江苏省房地产业协会

反馈：api-feedback@bzf.jiangsu.gov.cn · 工单系统 → 12345 政务直达
