# 系统初始化抽象层 · 地域/部门/业务词配置

> 目标：同一套页面，不止用于**江苏**，也可复制到**广西/贵州**等省份（含下属设区市维度）；
> 主管厅不止**住建厅**，也可换**商务厅**；业务词（保租房）随厅而变。
> 这两个维度 + 金融机构 + 国企持有方 + 生态机构示例名单，全部在 `screens/_region.js` 一处配置。

---

## 1 · 架构

```
screens/_region.js   ← 系统初始化层 · 单一数据源（预设注册表 PRESETS）
   │
   ├─ window.BZF_REGION   当前激活预设（含派生字段 channel/deptName/govOrg/bureau）
   ├─ window.BZF_PRESETS  全部预设（供 overview 切换下拉）
   │
   ├─(路径1) 导航/chrome 直接读 BZF_REGION 拼装：
   │         _nav.js（桌面 sidebar）· _navmobile.js（移动 chrome）· index.html / home.html（城市切换器）
   │
   └─(路径2) relabel()：DOMContentLoaded 时把"江苏基准串 → 激活预设值"
             按词典替换页面正文文本节点 ⇒ 70+ 页面正文【零改动】
```

**预设切换**（优先级高→低）：`?region=gx` 查询参数 → localStorage `bzf_region` → 默认 `js`。
`overview.html` 顶部有 🌐 预设下拉，切换即带 `?region=` 跳转。

**默认 `js`（江苏·住建厅）= 现有页面字面量基准**，切到它时 relabel 为空操作，零风险。

## 2 · 配置的 9 类主体名词

| # | 变量 | 江苏基准 | 配置字段 |
|---|---|---|---|
| 1 | 省（全称/简称/省会/设区市） | 江苏省 / 江苏 / 南京 / 13 市 | `prov.{full,short,capital,cities}` |
| 2 | 主管厅 | 住建厅 | `dept.short` |
| 3 | 厅组织全称词根 | 住房和城乡建设 / 住建 | `dept.{orgFull,stem}` |
| 4 | 处室 | 监管处 | `dept.division` |
| 5 | 业务词 | 保租房 / 保障性租赁住房 | `biz.{term,termFull}` |
| 6 | 金融机构 | 江苏银行 / 普惠金融部 | `bank.{name,div}` |
| 7 | 国企持有方 | 南京安居保障房集团 / 安居集团 | `soe.{full,mid,short}` |
| 8 | 生态机构示例名单 | 自如/龙湖/冠寓/华润/招商/省心租、优家装饰、北新建材、家协培训 | `roster.{brands,vendor,supplier,training}` |

**新增省份/厅**：只在 `PRESETS` 加一个 key，**不改任何页面**。

## 3 · relabel 词典规则

- 源串**始终是 `PRESETS.js`（江苏基准）**——所以页面正文仍以江苏字面量书写，保持基准可读。
- 按**源串长度降序**应用，长串优先，杜绝子串误替换（如先 `住房和城乡建设` 后 `住建`，先 `江苏省` 后 `江苏`）。
- 设区市按**下标一一对应**替换（省会在 index 0）。
- ⚠️ **不替换裸 `保障房`**：它是国企名 `安居保障房集团` 的子串，替换会误伤（实测发现并已规避）。标准全称走 `保障性租赁住房` 单独覆盖。
- 运营方品牌 `贝壳` 是跨域常量，仅极少数演示场景才换。

## 5 · 验证结论（node mock DOM）

- `js`：所有样例字符串与导航文案 **零改动** ✓
- `gx`：`广西/广西北部湾银行/南宁市住建局/南宁安居保障房集团`，仍住建厅、仍保租房 ✓
- `gz_zj`：`住建厅监管端/贵州住建住宿/贵阳市住建局/贵州银行/住建住宿通道/贵阳住建集团`，运营商换为 `华住/锦江都城/首旅/亚朵/贝壳省心住` ✓

## 6 · 已知边界（非本层职责）

1. **relabel 只处理静态文本节点 + `<title>`**：页面加载后由 JS 动态生成的含品牌串内容不会被替换 —— 这类内容应改读 `BZF_REGION`。
2. **目标省设区市少于江苏 13 个**时，多出的江苏市名会在正文样例（如监管大屏的城市表/热力图，最多 13 行）里残留。已用 `prov.extraCities` 兜底：贵州 9 个地级行政区之外，补 4 个县级市（仁怀/清镇/赤水/盘州）吸收江苏第 10-13 市，**杜绝江苏市名残留**；城市切换器仍只读 `prov.cities`（9 个），显示"9 设区市"。新增省份若市数 <13，应同样补 `extraCities` 到 13 个。贵州 3 个自治州用全称（黔东南苗族侗族自治州等）。
3. 需保持江苏原值的样例节点，加 `data-noregion` 跳过。
4. **改名 ≠ 改业务语义**：把保租房换成旅居住宿后，F 系列公积金/监管账户流程、好房子四维评级口径等是**业务重做**，需单列工作量。
5. **试点定位**（CLAUDE.md 规则 5）：江苏=验证基准；广西/贵州预设属"政策推动+自愿复制后的上限假设"，非承诺。
